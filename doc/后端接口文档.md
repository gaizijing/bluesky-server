# 低空气象飞行保障服务系统 — 后端接口文档（V2）

> **技术栈**：Spring Boot 3 + PostgreSQL + Flyway + MyBatis-Plus  
> **Base URL**：`http://localhost:8080/api`  
> **在线文档**：`http://localhost:8080/api/doc.html`（Knife4j，推荐）  
> **数据来源详解**：[API-接口与数据来源.md](./API-接口与数据来源.md)

---

## 1. 通用规范

### 1.1 统一响应

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 1.2 认证

除 `/auth/login`、`/auth/register` 等白名单外，请求头携带：

```
Authorization: Bearer <JWT>
```

### 1.3 时间

- 请求时间参数：ISO-8601 或 `yyyy-MM-dd HH:mm:ss`（Asia/Shanghai）
- 业务时间桶：15 分钟对齐（`TimeBucketUtil.BUCKET_MINUTES = 15`）
- 响应中的 `TemporalMeta`：`requestedTime`、`bucketTime`、`computedAt`、`isStale`

### 1.4 Region 与起降点

V2 以 **Region**（`R1` 天津宁河、`R2` 青岛）为数据隔离单元；**起降点** ID 如 `LP001`。

原 V1「重点关注区域」`/monitoring-points` 已移除，请使用 `/regions` 与 `/landing-points`。

## 2. 接口索引

### 认证

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/login` | 登录 |
| POST | `/auth/logout` | 登出 |
| GET | `/auth/userInfo` | 当前用户 |

### Region / 起降点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/regions` | Region 列表 |
| GET | `/regions/default` | 默认 Region |
| GET | `/regions/{regionId}` | Region 详情 |
| POST | `/regions` | 创建 |
| PUT | `/regions/{regionId}` | 更新 |
| PUT | `/regions/{regionId}/default` | 设为默认 |
| DELETE | `/regions/{regionId}` | 删除 |
| GET | `/landing-points?regionId=` | 起降点列表 |
| GET | `/landing-points/{id}` | 起降点详情 |
| POST | `/landing-points` | 创建 |
| PUT | `/landing-points/{id}` | 更新 |
| DELETE | `/landing-points/{id}` | 删除 |

### 气象

| 方法 | 路径 | 主要参数 | 数据来源 |
|------|------|----------|----------|
| GET | `/weather/realtime` | `pointId`, `time?` | Open-Meteo 实况 |
| GET | `/weather/point` | `lng`, `lat`, `time?`, `heightM`, `products?`, `includeRisk?` | Open-Meteo |
| POST | `/weather/by-coords/batch` | `{ "coordinates": [{lng,lat}, ...] }` | Open-Meteo |
| GET | `/weather/forecast-trend` | `pointId` | Open-Meteo |
| GET | `/weather/grid-field` | `regionId`, `product?`, `time?`, `heightM` | 缓存 / 种子 |
| GET | `/weather/heatmap/citywide` | — | risk_field_cache + IDW |
| GET | `/weather/vertical-profile` | `landingPointId`, `startTime?`, `endTime?`, `heightLevelsM?` | 表 / 派生 |
| GET | `/meteorology/vertical-profile` | `pointId`, `timeType` | **已废弃**，同上 |

`product` 可选：`temperature` / `wind` / `visibility` / `precip` / `humidity` / `cloud` / `pressure`

### 风场

| 方法 | 路径 | 主要参数 | 数据来源 |
|------|------|----------|----------|
| GET | `/wind-field` | `regionId?`, `time?`, `heightM`, `bounds?` | NetCDF（NOAA Reanalysis2） |

### 风险

| 方法 | 路径 | 主要参数 |
|------|------|----------|
| GET | `/risk/heatmap` | `regionId`, `time?`, `heightM`, `west?`, `south?`, `east?`, `north?` |
| GET | `/risk/point` | `lng`, `lat`, `time?`, `heightM` |

### 适飞

| 方法 | 路径 | 主要参数 |
|------|------|----------|
| GET | `/flyability/landing-matrix` | `regionId`, `landingPointId?`, `time?`, `hours` |
| GET | `/flyability/route-matrix` | `regionId`, `routeId`, `routeVersionId?`, `time?`, `hours` |

适飞等级：`GREEN` / `YELLOW` / `RED`

### 预警

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/warnings` | 列表（`regionId`, `types?`, `statuses?`, `limit?`） |
| GET | `/warnings/{warningId}` | 详情 |
| POST | `/warnings/{warningId}/ack` | 标记已读 |
| POST | `/warnings/{warningId}/handle` | 处理 |
| POST | `/warnings/{warningId}/close` | 关闭 |

状态流转：`NEW` → `ACKNOWLEDGED` → `HANDLED` / `CLOSED`

### 航路

## 3. 关键接口示例

### 3.1 登录

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'
```

### 3.2 实时气象

```bash
curl "http://localhost:8080/api/weather/realtime?pointId=LP001" \
  -H "Authorization: Bearer <token>"
```

### 3.3 气象格点

```bash
curl "http://localhost:8080/api/weather/grid-field?regionId=R1&product=wind&heightM=100" \
  -H "Authorization: Bearer <token>"
```

缓存未命中时响应含 `cacheMiss: true`、`grid: []`。开发环境可先执行：

```bash
curl -X POST "http://localhost:8080/api/scheduler/recompute?regionId=R1" \
  -H "Authorization: Bearer <token>"
```

### 3.4 航线风险分析

```bash
curl -X POST "http://localhost:8080/api/routes/route-sf-huangdao/analyze" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"currentTime":"2026-06-11T08:00:00+08:00"}'
```

## 4. 数据来源速查

| 类型 | 接口示例 | 说明 |
|------|----------|------|
| 🟢 真实外部 | `/weather/realtime`, `/wind-field`, `/isim/*` | Open-Meteo / NetCDF / UDP |
| 🟡 混合 | `/weather/grid-field`, `/risk/*`, `/flyability/*`, `/routes/analyze` | 调度缓存 + API；或 Flyway 种子 |
| 🟠 演示 | `/warnings`, `/cameras/preview`, `/ai/conclusion` | 种子 / 占位 / 模板 |
| ⚪ 配置 | `/regions`, `/routes`, `/flight/tasks` | 纯 DB |

## 5. V1 → V2 迁移对照

| V1（已废弃） | V2 |
|--------------|-----|
| `/monitoring-points` | `/landing-points` + `/regions` |
| `/weather/microscale` | 无 HTTP；内部用 `risk_field_cache` |
| `/weather/wind-field` | `/wind-field` |
| `/weather/risk/report` | `/warnings` |
| `/suitability/status` | `/flyability/landing-matrix` |
| `/suitability/heatmap` | `/weather/grid-field` 或 `/risk/heatmap` |
| `/meteorology/core-indicators` | 无 V2 接口 |
